# Email Campaign Pipeline — Agent Context

This project is an AO workflow that automates end-to-end email campaign production.
Agents working in this repo should understand the following.

## What This Pipeline Does

Takes a campaign brief (`config/brief.json`) and subscriber list (`data/subscribers.csv`),
then produces a complete campaign package:
- Audience segmentation and messaging framework (campaign-strategist)
- 8 copy variants across 4 segments × 2 A/B variants (copywriter)
- Compliance validation against CAN-SPAM and GDPR (compliance-checker)
- Responsive HTML email templates (template-builder)
- Timezone-optimized send schedule (Python script)
- A/B test configuration with statistical sizing (Python script)
- Final campaign summary report (campaign-analyst)

## Directory Layout

```
config/          # Static config — brief, brand, compliance rules
data/            # Runtime intermediate data — strategy, compliance reports
output/
  copy/          # JSON copy variants per segment-variant
  templates/     # HTML email templates per variant
  schedule.json  # Per-timezone send schedule
  ab-test-config.json  # A/B test group assignments
  campaign-summary.md  # Final human-readable report
templates/       # Input templates — base HTML, copy guidelines
scripts/         # Python utility scripts
sample-data/     # Example configs for local testing
```

## Key Files

- `config/brief.json` — Campaign brief (INPUT — edit this to set up a new campaign)
- `config/brand.json` — Brand identity (colors, fonts, logo, footer)
- `config/compliance-rules.json` — CAN-SPAM/GDPR rules and spam trigger word list
- `data/subscribers.csv` — Subscriber list (segment, timezone, engagement rates)
- `data/campaign-strategy.json` — Generated by campaign-strategist (do not hand-edit)
- `data/compliance-report.json` — Generated by compliance-checker
- `data/compliance-feedback.json` — Rework instructions for copywriter (only exists if fix-required)
- `output/copy/manifest.json` — Index of all copy variants
- `output/campaign-summary.md` — Final campaign report (human-readable)

## Compliance Rules Summary

The compliance-checker enforces:
1. CAN-SPAM: physical address + unsubscribe reference in all emails
2. GDPR: consent language for EU subscribers
3. No spam trigger words in subject lines (see config/compliance-rules.json)
4. No ALL CAPS, max 2 exclamation points, max 5-word CTA button text

## A/B Test Design

Each segment has 2 variants (A and B). The test element varies by segment per the brief:
- Subject line test: different subject, same body/CTA
- CTA test: same subject/body, different CTA button text

Test groups are sized for statistical significance using normal approximation.
Winner auto-sends after the configured test_duration_hours with 95% confidence threshold.

## Adding a New Campaign

1. Edit `config/brief.json` with the new campaign details
2. Update `data/subscribers.csv` with the latest subscriber export
3. Run: `ao workflow run email-campaign-pipeline`
4. Monitor: `ao daemon stream --pretty`
5. Review: open `output/templates/preview-index.html` in a browser
6. Upload templates and schedule to your ESP using `output/campaign-package.json`

## MCP Servers Used

- `filesystem` — all file I/O within this project directory
- `sequential-thinking` — structured strategy reasoning (plan-campaign phase only)

## Error Recovery

If compliance check fails with `fix-required`:
- Check `data/compliance-feedback.json` for specific revision instructions
- The copywriter agent will automatically revise flagged variants on the next run
- Max 3 rework attempts; if still failing after 3, review compliance rules and brief

If `blocked` verdict from compliance:
- Check `data/compliance-report.json` for the structural issue
- Usually caused by missing `sender.physical_address` in `config/brief.json`
- Fix the root cause and restart: `ao workflow run email-campaign-pipeline`
